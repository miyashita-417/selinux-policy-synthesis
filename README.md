電子情報通信学会 インターネットアーキテクチャ研究会 (IA) にて発表の論文「最小特権を考慮したSELinuxポリシーの制約に基づく自動合成」における検証用コードおよび補足資料のリポジトリです。



本研究では、運用意図を示す機能要件を論理制約へ変換し、Z3ソルバを用いて最小特権のSELinuxポリシーを自動合成する手法を提案しています。



\## リポジトリの構成

\- `src/target/mydaemon.c`: 検証に使用した仮想デーモンプロセス

\- `src/tool/solver.py`: SMTソルバを用いたポリシー自動合成ツール

\- `documents/constraints.md`: 論文紙面の都合上省略された機能要件および論理制約の詳細定義



\## 動作要件

\- Python 3.x

\- z3-solver

原稿の表2において、PID制御のdirクラスの具体的な権限に `rmdir` が含まれていますが、これは誤りです。
実際の結果には `rmdir` は含まれていません。

$read, lock, ...$

### 2. 原稿4.4(3)ネットワーク待ち受けに関するマクロ記述 の説明について
原稿4.4 (3)ネットワーク待ち受けに関するマクロ記述 について、
「このマクロは、ソケットと指定されたポートの紐付け権限に加え、TCPソケット全般に対する作成、読み書き、接続確立などの権限が包括的に付与される構造となっている。」という記述がありますが、これは誤りです。
正確な表現は以下の通りです。
- **訂正:**   このマクロは、指定されたポートへの紐付け権限と、特権ポートを利用するケーパビリティを付与するものである。なお、TCPソケット全般の作成や読み書き、接続確立などの基本操作権限は、別途定義された$\\texttt{create\_stream\_socket\_perms}$を含むルールによって許可されている。

<br>
- 補足: ソケットの作成や読み書きなどの基本権限は、別途定義されている `create_stream_socket_perms` によって許可されています。
