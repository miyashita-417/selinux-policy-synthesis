電子情報通信学会 インターネットアーキテクチャ研究会 (IA) にて発表の論文「最小特権を考慮したSELinuxポリシーの制約に基づく自動合成」における検証用コードおよび補足資料のリポジトリです。



本研究では、運用意図を示す機能要件を論理制約へ変換し、Z3ソルバを用いて最小特権のSELinuxポリシーを自動合成する手法を構成しています。



\## リポジトリの構成

\- `src/target/mydaemon.c`: 検証に使用した仮想デーモンプロセス

\- `src/tool/solver.py`: SMTソルバを用いたポリシー自動合成ツール

\- `documents/constraints.md`: 論文紙面の都合上省略された機能要件および論理制約の詳細定義



\## 動作要件

\- Python 3.x

\- z3-solver

## 原稿に関する正誤表 (Errata)

### 1. 4.5 表2 提案手法と既存マクロにより付与される権限の比較 について
表2において、PID制御のdirクラスの具体的な権限に誤りがありました。訂正は以下の通りです。
<br>
- **誤:**    $read, rmdir, lock, ...$
- **正:**    $read, lock, ...$

### 2. 4.4(3)ネットワーク待ち受けに関するマクロ記述 の説明について
4.4 (3)ネットワーク待ち受けに関するマクロ記述 について、説明に誤りがありました。訂正は以下の通りです。
<br>
- **誤:**   このマクロは、ソケットと指定されたポートの紐付け権限に加え、TCPソケット全般に対する作成、読み書き、接続確立などの権限が包括的に付与される構造となっている。

- **正:**   このマクロは、指定されたポートへの紐付け権限と、特権ポートを利用するケーパビリティを付与するものである。なお、TCPソケット全般の作成や読み書き、接続確立などの基本操作権限は、別途定義された`create_stream_socket_perms`を含むルールによって許可されている。
